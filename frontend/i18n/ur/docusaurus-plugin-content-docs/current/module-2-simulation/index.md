# روبوٹ سمیولیشن (Robot Simulation)

ماڈیول 2 میں خوش آمدید۔ سمیولیشن (Simulation) روبوٹکس ڈیولپمنٹ کے لیے ضروری ہے - یہ آپ کو الگورتھم محفوظ طریقے سے ٹیسٹ کرنے، ٹریننگ ڈیٹا بنانے، اور مہنگے ہارڈویئر کے بغیر تیزی سے کام کرنے دیتی ہے۔

## سیکھنے کے مقاصد (Learning Objectives)

- سمجھیں کہ سمیولیشن روبوٹکس کے لیے کیوں اہم ہے
- Gazebo سمیولیشن انوائرنمنٹ سیٹ اپ کریں
- URDF کے ساتھ روبوٹ ماڈلز بنائیں
- ROS2 کو سمیولیٹڈ روبوٹس کے ساتھ انٹیگریٹ کریں

## سمیولیشن کیوں؟ (Why Simulate?)

سمیولیشن اہم فوائد فراہم کرتی ہے:

| فائدہ | تفصیل |
|---------|-------------|
| **حفاظت (Safety)** | خطرناک منظرناموں کو بغیر خطرے کے ٹیسٹ کریں |
| **لاگت (Cost)** | بگز سے ہارڈویئر کا نقصان نہیں |
| **رفتار (Speed)** | ریئل ٹائم سے تیز چلائیں |
| **ڈیٹا (Data)** | لامحدود ٹریننگ ڈیٹا بنائیں |
| **متوازی کاری (Parallelization)** | سینکڑوں انسٹینسز چلائیں |

### سمیولیشن سے حقیقت کا فرق (Sim-to-Real Gap)

حقیقی روبوٹ سمیولیٹڈ روبوٹس سے مختلف ہوتے ہیں:

```text
┌─────────────────────────────────────────────────────────┐
│                    Sim-to-Real Gap                       │
├─────────────────────────────────────────────────────────┤
│  سمیولیشن           │          حقیقت                    │
│  ─────────────      │          ───────                  │
│  • مکمل سینسرز      │  • شور، ڈرفٹ، ناکامیاں           │
│  • مثالی فزکس      │  • رگڑ، لچک                       │
│  • معلوم حالت      │  • جزوی مشاہدہ                   │
│  • یقینی           │  • سٹوکاسٹک اثرات               │
└─────────────────────────────────────────────────────────┘
```

ڈومین رینڈمائزیشن (Domain Randomization) اور سسٹم آئیڈینٹیفکیشن اس فرق کو کم کرنے میں مدد کرتے ہیں۔

## Gazebo کا جائزہ

**Gazebo** سب سے مقبول اوپن سورس روبوٹکس سمیولیٹر ہے:

- **فزکس انجنز (Physics Engines)**: ODE, Bullet, DART, Simbody
- **سینسر سمیولیشن (Sensor Simulation)**: کیمرے، LiDAR, IMU, GPS
- **ROS2 انٹیگریشن**: gazebo_ros کے ذریعے مقامی سپورٹ
- **ورلڈ بلڈنگ (World Building)**: ماحول بنانے کے لیے GUI ٹولز

## Gazebo سیٹ اپ کرنا

### انسٹالیشن

```bash
# ROS2 Humble کے لیے Gazebo انسٹال کریں
sudo apt install ros-humble-gazebo-ros-pkgs

# انسٹالیشن کی تصدیق کریں
gazebo --version
```

### ROS2 کے ساتھ Gazebo چلانا

```bash
# خالی ورلڈ شروع کریں
ros2 launch gazebo_ros gazebo.launch.py

# مخصوص ورلڈ کے ساتھ شروع کریں
ros2 launch gazebo_ros gazebo.launch.py world:=/path/to/world.sdf
```

### Gazebo GUI

Gazebo انٹرفیس فراہم کرتا ہے:

- **3D ویو**: سمیولیشن ورلڈ دیکھیں
- **ماڈل پینل**: آبجیکٹس اور روبوٹس داخل کریں
- **ورلڈ پینل**: فزکس اور لائٹنگ ایڈجسٹ کریں
- **ٹائم لائن**: سمیولیشن ٹائم کنٹرول کریں

## بنیادی ورلڈ بنانا

رکاوٹوں کے ساتھ سادہ ورلڈ بنائیں:

```xml title="simple_world.sdf"
<?xml version="1.0" ?>
<sdf version="1.6">
  <world name="simple_world">
    <!-- زمینی سطح -->
    <include>
      <uri>model://ground_plane</uri>
    </include>

    <!-- سورج کی روشنی -->
    <include>
      <uri>model://sun</uri>
    </include>

    <!-- باکس رکاوٹ -->
    <model name="box">
      <pose>2 0 0.5 0 0 0</pose>
      <link name="link">
        <collision name="collision">
          <geometry>
            <box><size>1 1 1</size></box>
          </geometry>
        </collision>
        <visual name="visual">
          <geometry>
            <box><size>1 1 1</size></box>
          </geometry>
          <material>
            <ambient>0.8 0.2 0.2 1</ambient>
          </material>
        </visual>
      </link>
    </model>
  </world>
</sdf>
```
